Token System Design

3-tier architecture in Tailwind v4 @theme — Primitive → Semantic → Component

Tier 1 — Primitive
Raw values. No meaning. Full scale. Never used directly in components.
--color-blue-500: #3b82f6 --color-slate-900: #0f172a --size-4: 1rem --radius-md: 8px --font-sans: 'Inter'
↓ referenced by
Tier 2 — Semantic
Intent-based aliases. Light/dark variants defined here. What components consume.
--color-bg: var(--color-white) --color-brand: var(--color-blue-600) --color-text-primary: var(--color-slate-900) --color-border: var(--color-slate-200) --color-danger: var(--color-red-500)
↓ referenced by
Tier 3 — Component
Component-scoped tokens. Enables per-component theming without touching semantics.
--button-bg: var(--color-brand) --button-radius: var(--radius-md) --input-border: var(--color-border) --badge-radius: var(--radius-full)
